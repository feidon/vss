# CI and Pre-commit Linter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add pre-commit hooks (ruff, import-linter, eslint, prettier) and a GitHub Actions CI pipeline (lint + test + build) to the VSS monorepo.

**Architecture:** Single `.pre-commit-config.yaml` at repo root with directory-scoped hooks. Single `.github/workflows/ci.yml` with five parallel jobs filtered by path. No changes to application code.

**Tech Stack:** pre-commit, ruff, import-linter (Python); angular-eslint, prettier (Angular); GitHub Actions

**Spec:** `docs/superpowers/specs/2026-04-02-ci-and-pre-commit-design.md`

### Task 1: Add ruff to backend

**Files:**
- Modify: `backend/pyproject.toml`

- [ ] **Step 1: Install ruff as dev dependency**

Run from `backend/`:
```bash
uv add --dev ruff
```

This adds `ruff` to `[dependency-groups] dev` in `pyproject.toml` and updates `uv.lock`.

- [ ] **Step 2: Add ruff config to pyproject.toml**

Add to `backend/pyproject.toml`:
```toml
[tool.ruff]
target-version = "py314"
line-length = 88
src = ["api", "application", "domain", "infra", "tests"]

[tool.ruff.lint]
select = ["E", "F", "I", "UP"]
```

- `E` + `F`: pycodestyle + pyflakes (core errors)
- `I`: isort (import sorting)
- `UP`: pyupgrade (modern Python syntax)

- [ ] **Step 3: Run ruff to verify it works**

```bash
cd backend && uv run ruff check .
```

Expected: either clean output or a list of fixable issues. If issues found, run `uv run ruff check --fix .` then `uv run ruff format .` to auto-fix.

- [ ] **Step 4: Run ruff format to verify**

```bash
cd backend && uv run ruff format --check .
```

If files need formatting, run `uv run ruff format .` to fix them.

- [ ] **Step 5: Commit**

```bash
git add backend/pyproject.toml backend/uv.lock
git commit -m "chore: add ruff linter to backend dev dependencies"
```

If any files were auto-fixed, include them too:
```bash
git add -u backend/
git commit -m "style: auto-fix ruff lint and format issues"
```

### Task 3: Add pre-commit to backend and create config

**Files:**
- Modify: `backend/pyproject.toml`
- Create: `.pre-commit-config.yaml`

- [ ] **Step 1: Install pre-commit as backend dev dependency**

Run from `backend/`:
```bash
uv add --dev pre-commit
```

- [ ] **Step 2: Create `.pre-commit-config.yaml` at repo root**

Create `/home/feidon/Documents/vss/.pre-commit-config.yaml`:
```yaml
repos:
  # --- Backend: Ruff ---
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.12.2
    hooks:
      - id: ruff
        args: [check, --fix]
        types_or: [python, pyi]
        files: ^backend/
      - id: ruff-format
        types_or: [python, pyi]
        files: ^backend/

  # --- Backend: import-linter ---
  - repo: local
    hooks:
      - id: lint-imports
        name: lint-imports
        entry: bash -c 'cd backend && uv run lint-imports'
        language: system
        pass_filenames: false
        files: ^backend/.*\.py$

  # --- Frontend: ESLint ---
  - repo: local
    hooks:
      - id: eslint
        name: eslint
        entry: bash -c 'cd frontend && npx eslint src/'
        language: system
        pass_filenames: false
        files: ^frontend/src/.*\.(ts|html)$

  # --- Frontend: Prettier ---
  - repo: local
    hooks:
      - id: prettier
        name: prettier
        entry: bash -c 'cd frontend && npx prettier --check "src/**/*.{ts,html,css}"'
        language: system
        pass_filenames: false
        files: ^frontend/src/.*\.(ts|html|css)$
```

Note: Check the latest `ruff-pre-commit` release tag at https://github.com/astral-sh/ruff-pre-commit/releases and update `rev` accordingly.

- [ ] **Step 3: Install the git hooks**

Run from repo root:
```bash
cd backend && uv run pre-commit install
```

- [ ] **Step 4: Test pre-commit runs against all files**

```bash
cd backend && uv run pre-commit run --all-files
```

Expected: all hooks pass (or show expected issues if code hasn't been fully formatted yet — these should have been fixed in Tasks 1-2).

- [ ] **Step 5: Commit**

```bash
git add .pre-commit-config.yaml backend/pyproject.toml backend/uv.lock
git commit -m "chore: add pre-commit hooks for backend and frontend"
```

### Task 4: Create GitHub Actions CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the workflow file**

Create `/home/feidon/Documents/vss/.github/workflows/ci.yml`:
```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  changes:
    name: Detect Changes
    runs-on: ubuntu-latest
    outputs:
      backend: ${{ steps.filter.outputs.backend }}
      frontend: ${{ steps.filter.outputs.frontend }}
    steps:
      - uses: actions/checkout@v4
      - uses: dorny/paths-filter@v3
        id: filter
        with:
          filters: |
            backend:
              - 'backend/**'
            frontend:
              - 'frontend/**'

  backend-lint:
    name: Backend Lint
    needs: changes
    if: needs.changes.outputs.backend == 'true'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: backend
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v6

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.14"
          allow-prereleases: true

      - name: Install dependencies
        run: uv sync

      - name: Ruff check
        run: uv run ruff check .

      - name: Ruff format check
        run: uv run ruff format --check .

      - name: Import linter
        run: uv run lint-imports

  backend-test:
    name: Backend Test
    needs: changes
    if: needs.changes.outputs.backend == 'true'
    runs-on: ubuntu-latest
    env:
      DATABASE_URL: postgresql+asyncpg://vss:vss@localhost:5432/vss_test
    defaults:
      run:
        working-directory: backend
    services:
      postgres:
        image: postgres:17
        env:
          POSTGRES_USER: vss
          POSTGRES_PASSWORD: vss
          POSTGRES_DB: vss
        ports:
          - 5432:5432
        options: >-
          --health-cmd="pg_isready -U vss"
          --health-interval=10s
          --health-timeout=5s
          --health-retries=5
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v6

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.14"
          allow-prereleases: true

      - name: Install dependencies
        run: uv sync

      - name: Create test database
        run: PGPASSWORD=vss psql -h localhost -U vss -c "CREATE DATABASE vss_test;"

      - name: Run all tests
        run: uv run pytest -m '' -v

  frontend-lint:
    name: Frontend Lint
    needs: changes
    if: needs.changes.outputs.frontend == 'true'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: lts/*
          cache: npm
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: ESLint
        run: npx ng lint

      - name: Prettier check
        run: npx prettier --check "src/**/*.{ts,html,css}"

  frontend-test:
    name: Frontend Test
    needs: changes
    if: needs.changes.outputs.frontend == 'true'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: lts/*
          cache: npm
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npx ng test --no-watch

  frontend-build:
    name: Frontend Build
    needs: changes
    if: needs.changes.outputs.frontend == 'true'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: lts/*
          cache: npm
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npx ng build
```
